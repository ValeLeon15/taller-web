Requerimientos Funcionales:

  1. Autenticación y Autorización:
    • El sistema debe permitir a los usuarios registrarse como arrendatarios o arrendadores.
    • Debe existir un proceso de inicio de sesión seguro para ambos tipos de usuarios.
    • Los usuarios deben tener roles definidos (arrendatario o arrendador) con permisos correspondientes.
  2. Gestión de Propiedades:
    • Los usuarios arrendatarios deben poder crear, editar, eliminar y actualizar propiedades disponibles para arrendamiento.
    • Se debe proporcionar una interfaz intuitiva para la gestión de propiedades que incluya la capacidad de cargar imágenes y detalles relevantes.
  3. Interacción entre Usuarios:
    • Los usuarios arrendatarios deben poder revisar y responder a las solicitudes de arrendamiento recibidas.
    • Los usuarios arrendadores deben poder buscar propiedades disponibles y solicitar arrendamientos.
    • Ambos tipos de usuarios deben poder comunicarse entre sí a través de un sistema de mensajería integrado.
  4. Gestión de Pagos:
    • El sistema debe permitir a los usuarios realizar y recibir pagos de manera segura.
    • Los usuarios arrendadores deben poder realizar el pago correspondiente al arrendamiento de una propiedad.
    • Los usuarios arrendatarios deben poder verificar los pagos recibidos y realizar seguimiento de los mismos.
  5. Calificaciones y Comentarios:
    • Los usuarios arrendatarios deben poder proporcionar calificaciones y comentarios sobre la experiencia de arrendamiento.
    • Los usuarios arrendadores deben poder calificar tanto las propiedades arrendadas como a los arrendatarios.

Requerimientos No Funcionales:

  1. Seguridad:
    • El sistema debe garantizar la seguridad de los datos sensibles de los usuarios, como información de pago y datos personales.
    • Se deben implementar medidas de seguridad para prevenir ataques de hackers y fugas de información.
  2. Usabilidad:
    • La interfaz de usuario debe ser intuitiva y fácil de usar para usuarios de todos los niveles de habilidad.
    • Se deben proporcionar instrucciones claras y ayuda contextual en caso de que los usuarios encuentren dificultades.
  3. Rendimiento:
    • El sistema debe ser capaz de manejar un alto volumen de usuarios y transacciones simultáneas sin experimentar retrasos significativos.
    • Se deben realizar pruebas de carga para garantizar que la plataforma sea capaz de escalar adecuadamente.
  4. Disponibilidad:
    • El sistema debe estar disponible las 24 horas del día, los 7 días de la semana, con un tiempo de inactividad mínimo para mantenimiento programado.
    • Se deben implementar medidas de redundancia y respaldo para garantizar la continuidad del servicio en caso de fallos.
  5. Escalabilidad:
    • La arquitectura del sistema debe ser modular y escalable para poder adaptarse a futuros cambios en la demanda y funcionalidades adicionales.
    • Se debe planificar una estrategia de escalabilidad para garantizar que el sistema pueda crecer según sea necesario sin comprometer el rendimiento.
